# AgentBuff Co-Founder — PRD Implementation Backlog

> Generated from a rigorous multi-agent PRD-vs-code audit. **273 of 298 requirements are partial/missing.** No MVP shortcuts — every item must be implemented to PRD.

## Honest assessment
Honest assessment: the project is roughly 25-30% of the PRD-defined product. What is genuinely done and trustworthy is the deterministic core — the Financial Engine (HPP/BEP/margin/projections/loan/tax/ROI, 100% covered), the ValidationScore engine, the BYOK envelope crypto primitive, the Provider Abstraction Layer, and three adapter methods (validateCredential / generateStructured / groundedSearch) for Gemini and OpenAI — plus a working financial calculator UI and an in-process MCP tool registry. Almost everything else is scaffold or stub: Deep Research, image generation, vision and document-understanding adapter methods all throw 501; Brand Forge and Deck & Docs are static mockups with no service, no persistence, and no PDF renderer; the MCP Gateway has no HTTP transport, no JSON-RPC, no OAuth, and only 6 of 12 tools; there is no async job system (no Redis/BullMQ — AI calls run synchronously and block the UI for minutes); no object storage; no usage/cost tracking, audit log, or analytics despite the schema tables existing; no settings/key-management or credential-revocation flow; no Prisma migrations; and pervasive trust gaps (no schema validation of LLM output, no retry/backoff, no inline citation mapping, no prompt-injection isolation, no HTML sanitization before PDF, insecure auth-secret fallback, KEK auto-generated per-process risking silent data loss). The data model is well-designed but half of it (BrandKit, Document, UsageEvent, McpClient, McpAuditLog, PartnerFederation, plus several Project/Report columns) is dead schema never read or written. Net: the hard, differentiated trust foundation is solid, but the actual user-facing product (research pipeline, brand studio, document engine, agent gateway, jobs, storage, analytics) is largely unbuilt.
